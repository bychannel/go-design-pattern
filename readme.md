# go-design-pattern

设计模式的 golang 版本实现

* 理解设计原则，以及设计模式的使用场景和优缺点
* 设计模式是通用并经过实践的代码设计规范，重在学习其设计思想
* 掌握设计模式可以帮助我们阅读源码，利于构建大型的应用程序
* 设计模式的思想是相通的，切勿生搬硬套
* Go提倡面向接口编程，面向对象语言中的某些设计模式的写法在 Go 中会十分的别扭

**最后，设计模式不是银弹，也没必要处处使用设计模式，不要过早优化，持续重构才是正道**
